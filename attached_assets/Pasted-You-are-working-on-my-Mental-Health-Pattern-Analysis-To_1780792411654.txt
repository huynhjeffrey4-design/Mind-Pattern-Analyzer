## Task: Write backend tests for viewing and deleting journal entries

Focus only on journal entry tests. Do not refactor unrelated files. Do not add new features unless required to make these tests pass.

### Tests to add

Add tests for the journal API covering:

1. **Viewing all journal entries**

   * Create/authenticate a test user.
   * Create at least two journal entries for that user.
   * Call `GET /api/journals`.
   * Assert the response status is `200`.
   * Assert the response is a list.
   * Assert both created entries appear in the response.
   * Assert entries include expected fields like:

     * `id`
     * `content`
     * `sentiment_score`
     * `detected_keywords`
     * `created_at`

2. **Viewing one journal entry**

   * Create/authenticate a test user.
   * Create one journal entry.
   * Call `GET /api/journals/{journal_id}`.
   * Assert the response status is `200`.
   * Assert the returned journal entry has the correct `id` and `content`.

3. **Deleting a journal entry**

   * Create/authenticate a test user.
   * Create one journal entry.
   * Call `DELETE /api/journals/{journal_id}`.
   * Assert the response status is `200` or `204`, depending on the current API design.
   * Then call `GET /api/journals/{journal_id}`.
   * Assert the deleted journal entry is no longer accessible and returns `404`.

4. **User isolation for viewing journal entries**

   * Create two users.
   * Create a journal entry for User A.
   * Authenticate as User B.
   * Call `GET /api/journals/{user_a_journal_id}`.
   * Assert User B cannot view User A’s journal entry.
   * Expected status should be `404` or `403`, depending on the current API design.

5. **User isolation for deleting journal entries**

   * Create two users.
   * Create a journal entry for User A.
   * Authenticate as User B.
   * Call `DELETE /api/journals/{user_a_journal_id}`.
   * Assert User B cannot delete User A’s journal entry.
   * Expected status should be `404` or `403`.
   * Authenticate as User A and confirm the journal entry still exists.

6. **Unauthenticated access protection**

   * Call `GET /api/journals` without a token.
   * Assert it returns `401` or the app’s existing unauthorized status.
   * Call `DELETE /api/journals/{journal_id}` without a token.
   * Assert it returns `401` or the app’s existing unauthorized status.
