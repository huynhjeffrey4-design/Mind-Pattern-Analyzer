# MVP Build Plan: Mental Health Pattern Analysis Tool

## Project Goal

Build an MVP of a mental health pattern analysis web app where a user can:

1. Register and log in.
2. Complete daily check-ins.
3. Write journal entries.
4. View a dashboard with mood, stress, sleep, and habit trends.
5. View simple generated insights based on their own data.

The MVP should avoid diagnosis. It should only provide pattern-based observations.

---

# Global Build Rules for Engineering LLM

## Task Rules

Each task should:

* Focus on one concern only.
* Have a clear start and end.
* Be testable before moving to the next task.
* Avoid adding extra features not requested in the task.
* Avoid large refactors unless the task explicitly asks for one.
* Keep code simple and readable.

## Safety Rule

The application must never output clinical diagnoses.

Bad:

```text
You have depression.
```

Good:

```text
Your recent entries mention sadness, tiredness, and low motivation more often than usual.
```

## MVP Tech Stack

Use this stack unless instructed otherwise:

```text
Frontend: React + TypeScript + Vite
Backend: FastAPI + Python
Database: PostgreSQL
ORM: SQLAlchemy
Auth: JWT
Charts: Recharts
Styling: Tailwind CSS
```

# Final MVP Acceptance Criteria

The MVP is complete when all of the following are true:

## Authentication

* User can register.
* User can log in.
* User can log out.
* Protected pages require authentication.

## Daily Check-Ins

* User can submit mood, stress, sleep, habits, workload, energy, and notes.
* User can view check-ins.
* Invalid values are rejected.

## Journaling

* User can submit journal entries.
* User can view journal entries.
* Empty journal entries are rejected.
* Basic sentiment and keyword analysis runs.

## Dashboard

* User can see total check-ins.
* User can see average mood.
* User can see average stress.
* User can see average sleep.
* User can see mood trend chart.
* User can see stress trend chart.
* User can see sleep-mood chart.

## Insights

* User can generate insights.
* User can view insight cards.
* App shows “more data needed” when there are fewer than 3 check-ins.
* App detects at least:

  * Sleep and mood pattern
  * Exercise and mood pattern
  * Workload and stress pattern
  * Repeated journal keywords

## Safety

* App never gives a diagnosis.
* App shows supportive crisis information if serious self-harm language is detected.
* Journal entry is still saved unless another product decision is made.

## Privacy

* User A cannot access User B's data.
* API keys are not exposed in frontend.
* Passwords are hashed.
* JWT is required for protected backend routes.
